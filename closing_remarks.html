Closing remarks

Results and future work

In this website we've gone through the development of various parts of the bonding machine.

  1. We started by defining the general master-slave architecture of the machine and we wrote a custom communication protocol suited for our needs (see here);
  2. We designed and built the ultrasonic driver PCB to drive the ultrasonic welding element (see here);
  3. We implemented a nested PI controller with trajectory profiling for the 4 motors, achieving reliable and smooth motion. We also designed and built custom PCBs for the controllers (see here);
  4. We implemented another PI controller to control the heating stage (see here);
  5. We built a custom microscope with a CMOS camera to be able to accurately position the machine head on the substrate (see here);
  6. We built a modular power supply (24V, 12V, 5V, 3.3V) by cascading multiple buck converters (see here)
  7. By manually assisting the machine, we were able to perform a first bond test (see here);
  8. We almost fully automatized the bonding procedure (head descent and raise, ultrasonic driver activation, solenoid actuation) and successfully managed to bond a test substrate (see here);
We are very satisfied with the results we achieved. However, there are a few things we need to implement and refine.
  1. We want to display the camera output onto a LCD screen, and we also want to be able to control the machine through a touchscreen. We're currently stuck as we haven't been able to reliably connect the external RAM to the MCU due to crosstalk and other parasitic effects caused by long interconnections (see here);
  2. We haven't yet designed a custom PCB for the MCU as its design is not yet finalized. Unforeseen changes requiring PCB redesign might appear during development, and we can't afford to make many test PCBs. We need to nail it on the first try (see here).
